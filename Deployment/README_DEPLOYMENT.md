# QoS Buddy Deployment Package

This folder contains the runnable local deployment for QoS Buddy. It packages the integrated dashboard, gateway, agent bridges, AI services, authentication, memory, reporting, and supporting agent workspaces.

## Included

- Docker Compose application stack under `qos-buddy/`.
- Dashboard, gateway, RAG, reporting, synthesis, and bridge services.
- Monitoring collector source and supporting AI agent folders.
- Keycloak realm import with local demo users and roles.
- Safe `.env.example` files.
- Health checks, restore notes, and demo documentation.
- Packaged model and data artifacts needed for the local demo.

## Excluded Intentionally

- Real `.env` files and credentials.
- Jira tokens or other private integration secrets.
- Docker runtime volumes in `docker-data/`.
- Local logs, caches, build outputs, and dependency folders.
- Live telemetry JSONL streams generated during runtime.
- Local runtime databases and ChromaDB or MLflow state generated by containers.

## Requirements

- Windows 10 or 11.
- Docker Desktop running in Linux container mode.
- PowerShell.
- Python 3.10 or newer.
- Ollama installed and running locally.
- Local model:

```powershell
ollama pull qwen2.5
```

## Run

From this deployment folder:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\START-HERE.ps1
```

Then open:

```text
http://localhost:3000
```

## Notes

- ChromaDB is the active vector memory for incidents and RAG context.
- Qdrant is not required by the current Compose stack.
- Ollama is used locally through `host.docker.internal:11434`.
- Keep real `.env` files local and do not commit secrets.
