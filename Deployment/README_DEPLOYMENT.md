# QoS Buddy Deployment Package

This folder contains the sanitized SSD project export for QoS Buddy.

QoS Buddy is a local AI network assurance platform with live monitoring, anomaly detection, prediction, diagnostics, RAG memory, optimization recommendations, Keycloak role-based access, auditability, chatbot support, and executive reporting.

## Included

- Docker Compose application stack under `qos-buddy/`.
- Dashboard, gateway, RAG, reporting, synthesis, and bridge services.
- Monitoring collector source and supporting AI agent project folders.
- Keycloak realm import with demo users and roles.
- Safe `.env.example` files.
- Documentation, restore notes, health checks, and marketing overview.

## Excluded Intentionally

- Real `.env` files and credentials.
- Jira tokens or other private integration secrets.
- Docker runtime volumes in `docker-data/`.
- Local logs, caches, build outputs, and dependency folders.
- Live telemetry JSONL streams and local runtime databases.
- Chroma/MLflow runtime state generated by containers.

## Requirements On Another Machine

- Windows 10/11.
- Docker Desktop running in Linux container mode.
- PowerShell.
- Python 3.10 or newer.
- Ollama installed and running locally.
- Local model available:

```powershell
ollama pull qwen2.5
```

## Run

From the deployment folder:

```powershell
Copy-Item .\qos-buddy\.env.example .\qos-buddy\.env
.\START-HERE.ps1
```

Then open:

```text
http://localhost:3000
```

## Notes

- ChromaDB is the active vector memory for incidents and RAG context.
- Qdrant is not required by the current SSD Compose stack.
- Ollama is used locally through `host.docker.internal:11434`.
- Keep real `.env` files local and do not commit secrets.
