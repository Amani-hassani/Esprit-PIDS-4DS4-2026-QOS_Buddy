# QoS Buddy Deployment Package

This folder contains the sanitized SSD project export for QoS Buddy.

Included:
- Docker Compose application stack under `qos-buddy/`
- Shell/dashboard source, gateway, RAG, reporting, synthesis, and bridge services
- Monitoring collector source and supporting agent project folders
- Documentation, restore notes, health checks, and marketing overview
- Safe `.env.example` files

Excluded intentionally:
- Real `.env` files and credentials
- Docker runtime volumes in `docker-data/`
- Local logs, caches, build outputs, and dependency folders
- Live telemetry JSONL streams and local runtime databases
- Chroma/MLflow runtime state generated by containers

To run the main SSD stack on another machine:

1. Install Docker Desktop and Ollama.
2. Pull or create the local Ollama model expected by the stack, for example `qwen2.5`.
3. Copy `qos-buddy/.env.example` to `qos-buddy/.env`.
4. Fill in local secrets only on that machine.
5. From `qos-buddy/`, run:

```powershell
docker compose up -d --build
```

Then open `http://localhost:3000`.
